# 컴플라이언스 헌장 — 법적 불변식과 금지 기능 목록

작성일: 2026-07-02 · 근거: 규제 전면 재점검(웹 조사, 법률 자문 아님) · 관련: BUSINESS_PLAN.md §9, PRIVACY_POLICY.md, TERMS.md
**이 문서의 불변식을 깨는 기능 요청·개선 제안은 효용과 무관하게 거절한다. 각 항목은 규제 지위를 바꾸는 스위치다.**

## 1. 절대 불변식 (코드·문서·운영 공통)

| # | 불변식 | 근거 | 위반 시 |
|---|---|---|---|
| 1 | **운영자는 자금을 수수·보관·이체·정산하지 않는다.** 선금은 기관→음식점 계좌 직접 이동, 앱은 기록만 | 전금법 §2(선불·PG 정의) — 자금 관여가 없으면 PG·에스크로 비해당 | 무등록 전금업: 3년 이하 징역/2천만원 이하 벌금 |
| 2 | **선금 기록은 해당 음식점 1곳에서만 유효하다.** 다점포 통용 잔액·포인트 금지 | 전금법 §28③1호: 사업주 동일 1개 가맹점 선불수단은 **금액 무관 등록 면제** | 가맹점 2개 이상이면 등록 대상(2024.9 개정으로 강화) |
| 3 | **발행·차감의 실질 주체는 음식점이다.** 운영자는 원장을 열람·수정·정산 개입할 수 없다(서버는 복호화 불가 암호문만) | 금융위·금감원 유권해석: 발행자 판단 = 실질적 발행·관리 주체 | 운영자가 잔액 중앙관리·정산하면 발행자로 해석될 위험 |
| 4 | **서버는 평문 개인정보를 보유하지 않는다**(zero-knowledge). 암호문·총액·인원수·해시·접속기록만 | 개보법 — 암호문도 개인정보로 전제하되, 이 구조를 안전조치 증빙으로 활용 | — |
| 5 | **전화번호는 담당자 경로에 존재하지 않는다** | 데이터 최소화(개보법 §3) | — |
| 6 | **처리 완료 데이터는 30일 후 자동 삭제**(TTL cron). 클라우드 백업은 사용자 갱신·삭제 가능 | 개보법 보유기간 최소화 | — |
| 7 | **2028년 사업자등록 전까지 완전 무료.** 광고·유료 기능·수익화 금지 | 무료 유지가 통신판매업·청탁금지법 등 다수 판정의 전제 | 수익화 시 전면 재검토 트리거 |
| 8 | **GPS·실시간 위치 미수집**(주소·상호 텍스트 검색만) | 위치정보법 비해당의 전제 | — |
| 9 | **광고성 정보 미발송** | 정보통신망법 §50 비해당의 전제 | — |

## 2. 금지 기능 목록 (요청이 와도 구현하지 않는다)

1. 결제·이체·송금·대리수납·정산 자동화·에스크로 (→ 불변식 1, 3)
2. 여러 음식점에서 쓰는 공용 잔액·포인트·상품권 (→ 불변식 2)
3. 운영자(서버)의 원장 수정·잔액 조정·복호화 기능 (→ 불변식 3, 4)
4. 담당자 경로의 전화번호·주민번호 등 추가 식별정보 수집 (→ 불변식 5)
5. 개인 단위 거래 데이터의 서버 측 평문 축적 (→ §4 데이터 축적 가이드)

## 4. 데이터 축적 가이드 (2028년까지 "법에 저촉되지 않는 한도")

**축적 가능**: 거래 총액·건수·인원수 집계, 시간대별 사용 통계, 음식점 상호·주소(인허가 공공데이터 유래), 기관명.
**축적 금지**: 개인 단위 정보 일체, batch/개인 해시값(솔트 여부 무관 — 재식별 소지), 음식점 대표자명, 담당자 개인 이메일 평문(해시만).
**주의**: 인원 1~2명 소규모 셀 통계는 재식별 위험 → 집계 단위 확대 또는 제외. 축적 목적("서비스 개선을 위한 비식별 통계")은 처리방침·약관에 공개.
