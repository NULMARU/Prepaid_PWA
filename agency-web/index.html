선입금대장 — 담당자 업로드

0. 서버 주소

중계 서버(Worker) 주소. 로컬 목 테스트 시 목 서버 주소.

1단계 · 본인 확인

아래에 기관 이메일을 입력해 인증번호를 받으세요. 명단 작성 전 한 번만 확인하면 됩니다.
기관 이메일(.go.kr 또는 .korea.kr)만 사용할 수 있습니다.

2단계 · 음식점 찾기

음식점 이름으로 검색해 선택하세요. 결과가 많으면 지역(구·동)을 함께 입력해 좁힐 수 있습니다. 아직 등록되지 않은 음식점은 검색 결과에 나타나지 않습니다.
없음

3단계 · 명단 만들기

아래에 직원명·선금금액이 담긴 CSV(엑셀 저장 파일)를 올려 주세요. 형식은 자동으로 확인됩니다.
항목(순서 고정): 음식점명, 기관명, 부서명, 직원명, 선금금액, 입금자명 · 엑셀에서 그냥 'CSV로 저장'한 파일도 그대로 올리시면 됩니다(인코딩 자동 인식).

4단계 · 확인

아래 부서별 합계가 실제 이체하실 금액과 일치하는지 마지막으로 확인하세요.

5단계 · 보내기

전송 방법을 고른 뒤, 제3자 제공 동의를 확인하면 안전하게 보내기가 시작됩니다.
잘 모르겠으면 바로 보내기를 사용하세요.
추천

바로 보내기

암호화된 명단을 서버를 거쳐 음식점 앱으로 보냅니다. 서버는 내용을 볼 수 없으며, 음식점이 받는 즉시 서버에서 삭제됩니다(받지 않으면 최대 3일 후 자동 삭제).

직접 전달

명단이 서버에 저장되지 않습니다. 암호화된 파일을 만들어 이메일·카카오톡 등으로 직접 전달하거나, 화면의 QR을 음식점 사장님이 앱으로 찍게 하세요.

이 서비스는 결제·이체 기능이 없으며 자금을 보관하지 않습니다. 명단은 브라우저에서 암호화되어 해당 음식점만 열람할 수 있고, 서버는 내용을 볼 수 없습니다. 전화번호는 수집하지 않습니다. 선입금의 발행·차감·잔액 관리의 주체와 책임은 각 음식점에 있습니다.
음식점 검색: 행정안전부 LOCALDATA 공공데이터 활용
관련 문서(초안 — 법률 검토 후 확정): 개인정보처리방침(초안) · 이용약관(초안)
