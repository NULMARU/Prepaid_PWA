# 변경 이력 (CHANGELOG)

이 문서는 선입금대장 PWA의 주요 변경 사항을 기록합니다.

## [1.0.0-beta.1] - 2026-06-04

첫 베타 테스트 공개 버전입니다. (이전 `2.x` 표기는 내부 개발 과정의 번호로,
첫 외부 공개에 맞춰 `1.0.0-beta.1`로 재정비했습니다.)

### 주요 기능 (Features)
- 부서별 직원 선입금 잔액 관리
- 식사 사용 시 서명 기반 차감
- 일자별/직원별 거래 이력 조회
- 데이터 백업/복원 (JSON)
- 장부 엑셀(CSV) 내보내기
- 4자리 PIN 잠금
- 초기 설정 마법사 (공공데이터 API 연동 가게 검색)
- 오프라인 지원 (서비스 워커 캐시)

### 안정성 개선 (Fixed / Changed)
- **복원 버그 수정**: 안드로이드 브리지 복원 콜백 `onRestoreData`가 JSON만 파싱하고
  실제 복원을 수행하지 않던 빈 스텁 문제를 해결했습니다. 파일 복원과 동일한 로직을
  `restoreFromParsed`로 공통화하여 두 경로가 일관되게 동작합니다.
- **데이터 보존**: 복원 데이터에 직원/거래가 있으나 백업 메타에 `setupComplete`가
  없을 때, 복원 후 초기 설정 마법사로 되돌아가 데이터에 접근하지 못하던 문제를
  방지하도록 `setupComplete`를 유지합니다.
- **가게검색 폴백**: 초기 설정의 공공데이터 가게 검색 API 호출이 실패하면(네트워크/오류)
  자동으로 직접 입력 모드로 전환하여 가게 등록을 계속 진행할 수 있습니다.
- **캐시 동기화**: 서비스 워커 캐시 이름을 앱 버전과 일치시켜 신버전이 정상 배포되도록
  하고, `manual.html`을 캐시 자산에 추가했습니다.
- **당겨서 새로고침 방지**: 안드로이드에서 화면을 아래로 당겼을 때 페이지가 새로고침되어
  비밀번호 화면으로 되돌아가던 현상을 막기 위해 `overscroll-behavior:none`을 적용했습니다.
  (앱 재시작 시 비밀번호를 다시 묻는 보안 동작 자체는 유지됩니다.)
- **최신 버전 보장**: 서비스 워커를 화면(HTML) 요청에 한해 네트워크 우선으로 변경하여,
  온라인 상태에서는 항상 최신 버전을 받고 오프라인일 때만 캐시로 폴백하도록 했습니다.
- **배포 자동화**: `GitHub Actions` 기반 Pages 배포 워크플로를 추가하여 `main` 변경 시
  자동으로 최신 버전이 게시되도록 했습니다.
