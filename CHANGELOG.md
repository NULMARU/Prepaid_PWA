# 변경 이력 (CHANGELOG)

이 문서는 선입금대장 PWA의 주요 변경 사항을 기록합니다.

## [1.0.0-beta.7] - 2026-06-30

다자간 확장(기관–음식점–직원) 1차: 음식점 앱(원장) 로컬 기반을 추가했습니다. 서버 연동(담당자 웹·중계 서버)은 2차로 분리합니다.

### 추가 (Added)
- 거래 무결성 해시 체인(`tx_hash`/`prev_hash`)을 도입하고, 설정의 `무결성 검증`에서 체인 재계산과 입금·사용·잔액 합계 대조를 제공합니다. (스펙 §4.3, §7)
- RSA-OAEP 2048 키페어를 최초 실행 시 생성하고, 개인키는 AES-GCM으로 래핑해 기기에 저장합니다. (스펙 §4.1)
- `개인키 백업`: 사용자 암호로 보호된 개인키 백업 파일을 저장·복원합니다(기기 분실 대비). (스펙 STEP 12)
- `잔액 문자 안내(SMS 옵션)`: 차감 후 기기 문자앱으로 잔액을 안내합니다(직원명 미포함). 번호는 직원 동의 시에만 암호화 저장하며 `전화번호 일괄 파기`로 정산 종료 시 삭제합니다. (스펙 STEP 10, §5.2)
- CSV 임포트에 동명이인 자동 보완(`a/b/c` 접미사)과 경고를 추가했습니다. (스펙 §2.4)

### 변경 (Changed)
- IndexedDB 스키마를 v3로 올렸습니다(비파괴 마이그레이션: 기존 직원·거래 무손실). 백업 형식도 v3로 올렸으며 v1/v2 복원과 호환됩니다.

## [1.0.0-beta.6] - 2026-06-17

Android PWA 설치 안정성과 데이터 삭제 위험 안내를 보강했습니다.

### 추가 (Added)
- Android 설치 안정성을 높이기 위해 `icons/icon-192.png`, `icons/icon-512.png` PNG 아이콘을 추가하고 manifest에 등록했습니다.
- 오프라인 캐시에 PNG 아이콘을 포함했습니다.
- Chrome 사이트 데이터, 캐시, 저장공간 삭제 전 `장부 안전 저장`을 해야 한다는 안내를 앱 화면과 매뉴얼에 추가했습니다.

## [1.0.0-beta.5] - 2026-06-16

설정 화면의 데이터 관리와 부서 관리 흐름을 정리했습니다.

### 변경 (Changed)
- 설정 화면의 중복 `전체 초기화` 구역을 제거하고 데이터 관리 안에 한 번만 표시되도록 정리했습니다.
- `장부 엑셀로 저장`과 `데이터 안전 저장`을 `장부 안전 저장` 버튼 하나로 통합했습니다.
- 기관 부서 불러오기에서 기관을 바꾸면 현재 기관명이 자동으로 갱신되도록 했습니다.
- 거래 이력 삭제 위험이 큰 `새 달 시작` 기능을 설정 화면에서 제거했습니다.
- 설정 화면 메뉴 순서를 가게, 부서, 직원, 데이터, 보안 순으로 재배치했습니다.

## [1.0.0-beta.4] - 2026-06-16

서울시청 및 서울시 25개 구청의 과 단위 부서 선택 기능을 추가했습니다.

### 추가 (Added)
- `agency-departments.json`에 기관별 과 목록을 분리해 향후 데이터 업데이트를 쉽게 했습니다.
- 초기 설정과 설정 화면에서 기관을 선택하고 필요한 과를 복수 선택할 수 있습니다.
- 앱 안에서 기관 부서 목록을 새로고침할 수 있습니다.

## [1.0.0-beta.3] - 2026-06-16

소규모 음식점 로컬 사용에 맞춘 배포 준비 버전입니다.

### 변경 (Changed)
- PIN 5회 실패 후 PIN만 삭제하던 흐름을 제거하고, 모든 로컬 데이터를 삭제하는 앱 초기화만 허용하도록 변경했습니다.
- 백업 포맷을 `schemaVersion: 2`와 `payload` 구조로 정리하고, 복원 전 데이터 검증을 추가했습니다.
- 초기 설정의 외부 가게 검색 기능을 제거하고 음식점 정보 직접 입력만 남겼습니다.
- 설정 화면에 `새 달 시작` 버튼을 노출했습니다.
- 취소된 사용 거래는 일자별 사용 합계에서 제외합니다.
- 기본 기관명과 부서 예시를 특정 기관이 아닌 일반 매장용으로 변경했습니다.

## [1.0.0-beta.1] - 2026-06-04

첫 베타 테스트 공개 버전입니다. (이전 `2.x` 표기는 내부 개발 과정의 번호로,
첫 외부 공개에 맞춰 `1.0.0-beta.1`로 재정비했습니다.)

### 주요 기능 (Features)
- 부서별 직원 선입금 잔액 관리
- 식사 사용 시 서명 기반 차감
- 일자별/직원별 거래 이력 조회
- 데이터 백업/복원 (JSON)
- 장부 엑셀(CSV) 내보내기
- 4자리 PIN 잠금
- 초기 설정 마법사
- 오프라인 지원 (서비스 워커 캐시)

### 안정성 개선 (Fixed / Changed)
- **복원 버그 수정**: 안드로이드 브리지 복원 콜백 `onRestoreData`가 JSON만 파싱하고
  실제 복원을 수행하지 않던 빈 스텁 문제를 해결했습니다. 파일 복원과 동일한 로직을
  `restoreFromParsed`로 공통화하여 두 경로가 일관되게 동작합니다.
- **데이터 보존**: 복원 데이터에 직원/거래가 있으나 백업 메타에 `setupComplete`가
  없을 때, 복원 후 초기 설정 마법사로 되돌아가 데이터에 접근하지 못하던 문제를
  방지하도록 `setupComplete`를 유지합니다.
- **초기 설정 폴백**: 초기 설정 중 가게 정보를 직접 입력해 가게 등록을 계속 진행할 수 있습니다.
- **캐시 동기화**: 서비스 워커 캐시 이름을 앱 버전과 일치시켜 신버전이 정상 배포되도록
  하고, `manual.html`을 캐시 자산에 추가했습니다.
- **당겨서 새로고침 방지**: 안드로이드에서 화면을 아래로 당겼을 때 페이지가 새로고침되어
  비밀번호 화면으로 되돌아가던 현상을 막기 위해 `overscroll-behavior:none`을 적용했습니다.
  (앱 재시작 시 비밀번호를 다시 묻는 보안 동작 자체는 유지됩니다.)
- **최신 버전 보장**: 서비스 워커를 화면(HTML) 요청에 한해 네트워크 우선으로 변경하여,
  온라인 상태에서는 항상 최신 버전을 받고 오프라인일 때만 캐시로 폴백하도록 했습니다.
- **배포 자동화**: `GitHub Actions` 기반 Pages 배포 워크플로를 추가하여 `main` 변경 시
  자동으로 최신 버전이 게시되도록 했습니다.
